🧪 Testeur StreamHub Cast Receiver

Sender de test (web) pour piloter le Custom Web Receiver sans passer par l'app Flutter. À ouvrir dans Google Chrome desktop (le bouton Cast n'apparaît que dans Chrome avec le SDK Sender).

1. Connexion

Laisser vide pour utiliser le récepteur média par défaut de Google (CC1AD845) — utile uniquement pour vérifier que la connexion Cast fonctionne, mais aucun de vos messages custom ne sera traité par ce récepteur générique. Renseignez votre propre App ID pour tester réellement receiver.js.
SDK non initialisé

2. Charger une vidéo LOAD_VIDEO

DRM (optionnel)
Headers HTTP manifeste/segments (JSON, optionnel)
Marqueur Skip Intro (optionnel)
Vidéo suivante dans la queue (optionnel — test enchaînement)

3. Contrôles de lecture

Le bouton "Passer l'intro" est affiché sur le receiver — comme si l'utilisateur ne pouvait pas (ou ne voulait pas) utiliser la télécommande.
La boîte verte apparaît automatiquement dès que le receiver signale le bouton visible (SKIP_INTRO_VISIBILITY), pour reproduire le vrai flux app : l'utilisateur confirme depuis le téléphone sans avoir à connaître le timestamp.

4. Pistes audio / sous-titres

Les listes se remplissent automatiquement à partir du dernier STATE_UPDATE reçu une fois une vidéo avec plusieurs pistes chargée.
